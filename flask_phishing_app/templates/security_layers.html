Security Layers

The dashboard does not trust a single model score. It builds a verdict from several layers so suspicious URLs can be explained, challenged, and revisited with context.

Core Principle
A phishing verdict should be explainable from multiple directions.
That includes model evidence, transport posture, rendered content, browser artifacts, and threat-feed correlation.

Model Layer

The trained ensemble produces the core probability and supports explainability through SHAP so the strongest features can be inspected.

Transport Layer

DNS and SSL checks help identify suspicious hosting, certificate gaps, or weak transport posture often seen in throwaway phishing sites.

Content Layer

HTML patterns, NLP extraction, and brand or urgency cues expose whether the page behaves like a credential lure or social-engineering asset.

Intel Layer

Threat feeds, sandbox artifacts, screenshots, and optional Ollama summaries add context that a security reviewer can understand quickly.

Fast Layer

URL structure

Domain shape, suspicious tokens, redirects, and lexical patterns are scored immediately.

DNS and SSL

Fast network checks surface obvious infrastructure risks without waiting on the full enrichment pipeline.

Initial NLP and headers

Basic content cues and HTTP header review provide quick evidence for the first analyst-facing response.

Enrichment Layer

SHAP explainability

Top feature impacts clarify why the model leaned toward malicious or benign classification.

Screenshot and sandbox

Rendered evidence helps confirm impersonation themes and lets the analyst inspect page behavior more safely.

Threat feeds and LLM summary

Slow external checks and narrative reasoning deepen the investigation record after the first answer is already visible.

Scoring Logic

The hybrid score blends ensemble output with heuristic layers instead of treating model probability as ground truth. That matters because phishing infrastructure often looks statistically normal in one dimension while failing badly in another.

Security headers are a good example: missing CSP, HSTS, COOP, CORP, or weak cookie flags do not prove phishing alone, but they raise the overall risk when the URL, content, and infrastructure already look questionable.
