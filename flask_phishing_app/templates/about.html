Platform Overview

About PhishScope

PhishScope is built to help an analyst move from a bare URL to an explainable risk decision quickly, without losing the slower evidence that makes the decision defensible.

Fast verdict Background enrichment Human-readable reasoning
Operating Model
Classifier first, investigation detail second.
The UI is designed to return a usable answer immediately, then deepen the record with richer evidence as slower checks complete.

What It Combines

The dashboard merges model output with DNS, SSL, header posture, content extraction, threat intelligence, sandbox preview, and explainability artifacts.

Why It Exists

A raw phishing probability is not enough for review or incident handling. The app turns the score into evidence an analyst can read, compare, and retain.

Who It Helps

The interface fits security students, SOC-style triage workflows, and local investigations where speed matters but traceability still matters more.

How The Analysis Flows

1. Intake

The app normalizes the URL, extracts structural signals, and prepares the feature set for the trained ensemble.

2. Fast Pass

The first response returns the model score plus the most useful DNS, SSL, content, and header indicators with minimal wait time.

3. Enrichment

SHAP, screenshots, sandbox rendering, threat-feed checks, and Ollama reasoning are added later so the record becomes more complete over time.

What The Analyst Gets

  • A hybrid risk score with model probability and contextual signals.
  • Saved investigation history and per-record notes.
  • Explainability data instead of a black-box verdict.
  • HTML preview and screenshot context for suspicious pages.
  • Exportable reporting for follow-up and documentation.

Access And Workflow

The app supports a dedicated sign-in flow with local credentials and optional Google sign-in. That keeps the investigation interface separate from authentication, while still allowing history ownership, note tracking, and cleaner auditability.

The overall design is intentionally analyst-facing: it favors readable summaries, layered context, and saved records over a minimal single-score demo page.
